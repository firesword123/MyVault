# MyVault 功能与实现文档

## 文档范围

本文档描述当前 `Tauri + React + Rust` 版本的 MyVault，而不是历史上的旧实现。项目目标是构建一个借用 Windows 文件管理语义的数据型桌面应用：目录就是目录，文件就是文件，系统文件夹行为明确且可见。

## 当前技术栈

- 前端：React 19 + TypeScript + Vite
- 桌面容器：Tauri 2
- 后端：Rust
- 图标：Lucide SVG

## 数据目录设计

运行后的数据默认落在可执行文件旁边的 `myvault-data/` 中：

```text
myvault-data/
  settings.json
  vault/
    notes/
      drafts/
      inbox/
      trash/
      <用户文件夹...>
```

### 系统文件夹语义

- `drafts/`
  - 新建笔记的初始落点
  - 用于自动保存草稿
- `inbox/`
  - 草稿提交后的默认目录
  - 恢复失败时的回退目录
- `trash/`
  - 软删除区域
  - 回收站中的笔记支持恢复、彻底删除、清空回收站

### 用户文件夹语义

- 位于 `notes/` 下
- 支持创建、重命名、删除
- 删除限制为“仅空文件夹可删除”
- 允许存在嵌套目录

## 笔记文件格式

笔记使用 UTF-8 文本文件存储，包含 frontmatter 和正文：

```md
---
id: note-123
title: Example
created_at: 1710000000
updated_at: 1710000100
original_folder_path: projects/demo
deleted_at: 1710000200
---
<p>Hello</p>
```

### 元数据说明

- `id`
  - 笔记稳定标识
  - 当前仍作为实际文件名的一部分使用
- `title`
  - 显示标题
- `created_at`
  - 创建时间，秒级 Unix 时间戳
- `updated_at`
  - 最近更新时间，秒级 Unix 时间戳
- `original_folder_path`
  - 回收站恢复时优先回放的原目录
- `deleted_at`
  - 软删除时间

## 关键实现说明

### 软删除

- 删除笔记时不直接物理删除
- 后端把笔记文件移动到 `trash/`
- 同时写入 `original_folder_path` 与 `deleted_at`

### 恢复

- 恢复时优先尝试回到原文件夹
- 原文件夹不存在时回退到 `inbox/`
- 恢复后清理回收站元数据

### 文件夹删除

- 系统文件夹不可删
- 仅允许删除不包含笔记文件的目录树
- 删除使用递归目录删除，以兼容仅含空子目录的目录树

### 编码

- Rust 后端显式使用 UTF-8 读写文本
- 文档、配置、笔记均按 UTF-8 解释
- PowerShell 查看文件时建议显式指定 `-Encoding UTF8`
